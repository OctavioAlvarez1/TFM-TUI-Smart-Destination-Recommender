# Executive Dashboard Design

Document 13 – Executive Dashboard Design
TUI Smart Destination Recommender
Version 1.0

## Purpose

This document defines the Executive Dashboard strategy for the TUI Smart Destination Recommender.

The dashboard provides business stakeholders with a centralized view of tourism demand, sustainability performance, recommendation effectiveness, and congestion management outcomes.

The objective is to transform recommendation outputs into actionable business intelligence that supports strategic decision-making and responsible tourism initiatives.


## 1. Dashboard Objectives

The Executive Dashboard supports four primary objectives.

**Objective 1 – Business Visibility**

Provide a comprehensive view of recommendation system performance.

**Objective 2 – Sustainability Monitoring**

Measure how recommendations support sustainable tourism goals.

**Objective 3 – Congestion Management**

Monitor visitor distribution and overtourism risks.

**Objective 4 – Decision Support**

Enable business users to make informed operational and strategic decisions.


## 2. Target Users

The dashboard serves multiple stakeholder groups.

**Executive Leadership**

## 3. Dashboard Architecture

```
Data Sources
  │
  ▼
Data Layer (CSV files / BigQuery in production)
  │
  ▼
Executive Dashboard
  │
  ├── Business KPIs
  ├── Sustainability KPIs
  ├── Congestion KPIs
  ├── Recommendation KPIs
  │
  ▼

```

The architecture separates analytical reporting from operational systems.


## 4. Dashboard Sections

The dashboard consists of four primary modules.

**Module 1 – Executive Overview**

Provides a high-level summary of platform performance.

**Module 2 – Recommendation Analytics**

Measures recommendation effectiveness.

**Module 3 – Sustainability Monitoring**

Tracks environmental objectives.

**Module 4 – Congestion Management**

Monitors tourism distribution.

---

**MVP Implementation Note**

The MVP implements the dashboard concept through two components:

1. The **Analytics page** in the React frontend — the primary governance dashboard (Layer 5 of the 5-layer architecture). It features a glassmorphism hero with KPI cards, an interactive MonthlyChart (click any month bar to update all sections), a heat tile destination ranking with sidebar type filter (Beach/City/Nature/Mixed), a dynamic status breakdown (Overloaded/High Pressure/Moderate/Opportunity) that recalculates per selected month, and a full month-aware destination monitoring table.

2. The **Streamlit dashboard** (`src/dashboard/app.py`, accessible at `localhost:8501`) — an alternative read-only view of the same data, useful for rapid analytics exploration without the full React frontend.

For production, Looker or Power BI are the recommended reporting tools (see Section 12).

## 6. Recommendation Analytics Dashboard

Purpose: Measure recommendation engine performance.

KPIs:

**Recommendation Volume** — Number of recommendations generated.

**Top Recommended Destinations** — Most frequently recommended destinations.

**Recommendation Acceptance Rate** — Percentage of recommendations resulting in bookings.

**Average Recommendation Score** — Average ranking score generated by the engine.

**Recommendation Confidence** — Average recommendation confidence score.

Visualizations:
- Top Recommended Destinations: Bar Chart
- Recommendation Trends: Line Chart
- Recommendation Distribution: Histogram
- Confidence Score Distribution: Box Plot


## 7. Sustainability Dashboard

Purpose: Track environmental performance.

KPIs:

**Average Sustainability Score** — Measures overall recommendation sustainability.

**Sustainable Destination Ratio** — Percentage of recommendations directed toward highly sustainable destinations.

**Sustainability Impact Score** — Measures positive environmental contribution.

**Green Destination Adoption** — Percentage of bookings generated from sustainable destinations.

Visualizations:
- Sustainability Score Distribution: Histogram
- Sustainable vs Non-Sustainable Recommendations: Pie Chart
- Sustainability Trend: Line Chart
- Top Sustainable Destinations: Ranking Table

Sustainability Dashboard Layout:
```
-------------------------------------------------
|  Avg Sustainability  |  Green Adoption Rate      |
-------------------------------------------------
|  Sustainability Trend                           |
-------------------------------------------------
|  Sustainable Destinations Ranking              |
-------------------------------------------------
```


## 8. Congestion Management Dashboard

Purpose: Measure overtourism reduction effectiveness.

KPIs:

**Average Congestion Score** — Measures overall congestion exposure.

**High Congestion Destination Ratio** — Percentage of recommendations directed to highly congested locations.

**Diversification Rate** — Measures recommendation distribution across destinations.

**Congestion Reduction Impact** — Estimated reduction in overcrowded destination exposure.

Visualizations:
- Congestion Score Distribution: Histogram
- High-Risk Destinations: Heat Map
- Destination Diversification: Treemap
- Congestion Trends: Line Chart

Congestion Dashboard Layout:
```
-------------------------------------------------
|  Avg Congestion  |  Diversification Rate       |
-------------------------------------------------
|  Congestion Trend                              |
-------------------------------------------------
|  Destination Congestion Heatmap                |
-------------------------------------------------
```


## 9. Explainable AI Dashboard

Purpose: Provide visibility into recommendation logic.

KPIs:

**Recommendation Confidence** — Average confidence score.

**Sustainability Influence** — Percentage of rankings impacted by sustainability optimization.

**Congestion Influence** — Percentage of rankings adjusted by congestion balancing.

**Top Recommendation Drivers** — Most influential recommendation factors.

Visualizations:
- Feature Contribution Analysis: Bar Chart
- Recommendation Drivers: Ranking Table
- Confidence Distribution: Histogram
- Recommendation Explanation Examples: Interactive Panel

## 12. Technology Stack

Recommended implementation options.

**MVP — React Analytics Page (Built)**

The Analytics page in the React frontend (`frontend/src/pages/Analytics.tsx`) is the primary built implementation of the governance dashboard. It runs as part of the main SPA at `localhost:5173`.

**MVP — Streamlit (Built)**

`src/dashboard/app.py` provides an alternative read-only analytics view running at `localhost:8501`. Recommended for rapid exploration and hackathon demonstrations.

**Production Option 1 — Looker Studio**

Benefits:
- Native Google Cloud integration
- Rapid development
- Executive-friendly interface

Recommended for production deployment.

**Production Option 2 — Power BI**

Benefits:
- Enterprise reporting
- Advanced visualizations
